Linux interaction · least privilege · recovery

Recovery before novelty

A terminal aquarium can be playful; replacing part of a login or lock flow is still a systems change.

The Aquarian repository treats recovery as part of the feature. Its documented controls include preflight checks, user-level execution on TTY1, narrowly scoped privileges, an SSH recovery path, a shell fallback, and uninstall steps. The useful pattern is broader than the aquarium: when an interface sits near authentication or session control, the escape route is part of the design.

Evidence boundary: this note summarizes the public repository. The implementation was not reinstalled during this portfolio pass.

Wazuh · VirtualBox · host and guest agents

Make the monitoring topology legible

A local security lab becomes easier to debug when the appliance, host, guests, and agent-enrollment paths are documented as separate layers.

The Wazuh project records a 4.14.4 OVA in VirtualBox on macOS, resource tuning for the appliance, and enrollment paths for macOS and Debian agents. That separation matters: server reachability, virtual networking, endpoint enrollment, and dashboard access can fail independently, so they should be checked independently.

Safe-evidence rule: configuration patterns may be public; working credentials and sensitive endpoint details should not be repeated in portfolio copy.

storage · encryption · installation safety

Destructive instructions need visible risk

A technically complete storage guide is not responsible if the irreversible step is easier to miss than the happy path.

The Standalone Kali procedure covers physical-drive selection, BTRFS, LUKS, and boot configuration. Its most important constraint is not a command—it is certainty about the target device. Hardware compatibility is not universal, and a reader must validate the drive, firmware, backups, and current upstream guidance before acting.

Documented test context: the repository includes a 2015 Mac context. That is evidence of one setup, not a universal compatibility claim.
